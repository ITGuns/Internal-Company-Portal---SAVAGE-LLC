# Product

## Register

product

## Users

MyDeskii is used by SAVAGE LLC employees, managers, operations staff, payroll reviewers, admins, web developers, and client users. Internal users need fast operational control. Client users need confidence: what changed, what is waiting on them, what is planned next, and how to communicate with the team.

## Product Purpose

MyDeskii is an operating portal for daily work coordination and client delivery. It should help internal users manage tasks, logs, payroll, files, chat, approvals, and client operations. The client-side experience should feel like a polished delivery command center: progress, requests, updates, reports, resources, approvals, and next actions should be visible without exposing private internal production details.

## Current Remodel Direction

The client portal should move from a simple admin-style page stack into an executive client operations cockpit. The screenshots provided are design inspiration, not copy targets. Extract the useful patterns:

- A strong first viewport that immediately communicates status, value, and next action.
- Dashboard-style visual storytelling with a hero summary, metric strip, progress panel, and activity feed.
- High-contrast surfaces used deliberately for important summaries, not as a full-page gimmick.
- Reusable card and panel systems with consistent icons, numbers, labels, borders, and action placement.
- Clear process framing: request, review, build, update, approve, report.
- A confident CTA area for submitting a request, reviewing approvals, or contacting the team.

## Brand Personality

Practical, precise, confident, and premium. The interface should feel dependable for daily work and impressive enough for a paying client to trust the operation behind it. Polish should come from layout discipline, typography, spacing, state clarity, and purposeful visual contrast.

## Anti-references

- Generic admin tables with no client-facing story.
- Marketing landing pages copied directly into task-heavy product screens.
- Visual effects that slow repeated use or hide the next action.
- Purple/blue AI-style gradients, decorative glow spam, pure black, and pure white.
- Nested cards, default glassmorphism, identical card grids, and random decorative badges.
- Modal-first workflows when inline panels, drawers, or contextual actions keep more context visible.
- Fake metrics, generic placeholder names, dead links, and copy that says value without showing workflow.

## Design Principles

- Put the communication loop first: client request, team response, visible progress update, client approval, next action.
- Make status glanceable: ownership, progress, blockers, deadlines, freshness, and required client input should be visible quickly.
- Use executive-level hierarchy: one clear page story, then supporting panels and records.
- Keep product speed: frequent workflows need predictable controls, tactile feedback, and minimal waiting.
- Keep access boundaries obvious: client-visible content, internal-only notes, approvals, payroll, and admin actions must never blur.
- Design mobile as a real client review surface, not a shrunken desktop dashboard.
- Reuse shared shells, panels, tokens, and copy rules before route-specific styling.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve keyboard navigation, semantic landmarks, visible focus states, responsive layouts, reduced-motion behavior, and clear empty, loading, error, disabled, and success states. Premium visual treatment cannot reduce readability or hide critical workflow state.
